# CodeX Agent README
## 简介
CodeX Agent 是运行在 Obsidian 中的智能协作助手，负责在本地笔记库与代码仓之间建立一条可审计、可复用的自动化链路。它聚合上下文、执行命令、生成笔记产出，并帮助团队以“理解 -> 计划 -> 执行 -> 反馈”的闭环方式交付任务。

## 功能亮点
- **双向同步**：实时对齐 Obsidian 笔记与代码仓工作区，确保上下文一致。
- **智能计划**：内置 `codex plan` 流程，支持拆解任务、跟踪进度与阶段性交付。
- **安全执行**：遵守只读/沙箱策略，避免误删或覆盖用户改动，所有命令可审计。
- **上下文检索**：优先使用 `rg`、`git` 等高效命令获取项目现状，兼容多语言栈。
- **结构化输出**：响应中自动包含变更、行号、测试结果及后续建议，便于审阅。

## 系统架构
| 组件 | 说明 |
| --- | --- |
| Obsidian 笔记库 | 承载需求、输出和变更记录 |
| CodeX CLI | 命令执行核心，负责计划、日志、脚本、上下文管理 |
| 代码仓工作区 | 任意 Git 仓库，遵循只读/授权策略操作 |
| 辅助工具层 | `rg`、`git`、`node`、`python`、`pytest`、`npm` 等常用工具 |

## 环境要求
1. Windows 10/11，已安装 Obsidian 与 CodeX CLI；
2. 具备 `rg`、`git`、`node`/`npm`、`python`/`pip`、`pytest` 等工具（按项目需要）；
3. 命令行与 Obsidian 指向同一库根目录（例如 `<库根目录>`）；
4. 网络与权限符合组织策略（若受限需在输出中说明）。

## 安装与初始化
1. 安装 CodeX CLI（官方安装包或 `npm install -g @codex/cli`）；
2. `codex login` 绑定账户，确认 `codex --version` 正常；
3. 在终端进入库根目录（如 `cd <库根目录>`）；
4. 运行 `codex context sync`（若提供）同步上下文；
5. 使用 `codex note open <path>` 关联目标笔记。

## 常用命令速查
- `pwd` / `ls`：确认当前路径
- `rg --files`、`rg "pattern"`：检索代码/笔记
- `git status` / `git diff`：查看工作区与改动
- `apply_patch`：精确修改单文件
- `codex plan` + `codex plan update`：管理任务计划
- `npm test`、`pytest`、`go test ./...`：执行验证
- `codex log`：记录命令输出要点

## 推荐工作流程
1. **理解需求**：阅读笔记、运行 `rg`/`git` 获取上下文；
2. **制定计划**：复杂任务使用 `codex plan` 拆解步骤；
3. **执行变更**：遵循编辑规范，首选 `apply_patch`；
4. **验证结果**：运行对应测试或命令并记录结论；
5. **反馈总结**：输出修改点、行号、风险与下一步建议。

## 编辑规范
- 保持原有编码与格式，仅在必要处添加精炼注释；
- 不回滚或覆盖用户未授权的改动；
- 禁止使用 `git reset --hard`、`git checkout --` 等破坏性命令；
- 遇到权限受限或异常立即暂停并提示用户。

## 故障排查
- **命令失败**：阅读日志，必要时加 `--debug` 或使用替代工具；
- **工具缺失**：在输出中说明缺口并建议安装；
- **权限限制**：记录只读/无网络等客观原因；
- **CLI 异常**：尝试 `codex diag`/`codex doctor` 或检查日志目录。

## 最佳实践
- 进入仓库先 `pwd` + `git status` + `rg --files | head` 熟悉环境；
- 长流程任务保持阶段性交付，并在响应中附后续选项；
- 输出中引用文件时使用 `路径:行号` 形式，方便回溯；
- 结束前确认测试、计划状态与下一步提示都已更新。
